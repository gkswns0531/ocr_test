# Epic E: True Visual 분석 및 리포트

## 1. 배경 및 목적

SDS-KoPub-VDR-Benchmark의 `type=visual` 161건 중 실제로 이미지 정보가 필수인 쿼리를 선별하고, 인터랙티브 HTML 리포트로 시각적 분석을 수행하며, 검색 성공/실패 패턴을 정성적으로 분류하여 검색 전략 설계 근거를 도출.

## 2. True Visual 케이스 분류 (E-1)

### 2.1 분류 기준

- **true_visual**: 페이지 텍스트만으로는 검색이 불가능하고, 반드시 이미지/차트/그래프를 통해서만 검색할 수 있는 쿼리
- **not_visual**: `type=visual`로 레이블되었으나, 실제로는 텍스트 정보만으로 검색 가능한 쿼리

### 2.2 분류 결과

| 분류 | 건수 | 비율 |
|------|------|------|
| **true_visual (정성 분류)** | **58** | 36.0% (of 161 visual) |
| — GT 페이지에 image/chart region 존재 | **40** | 최종 평가 대상 |
| — GT 페이지에 image/chart region 없음 | 18 | region 검색이 구조적으로 불가 |
| not_visual | 103 | 64.0% (of 161 visual) |

`type=visual` 레이블의 64%가 실제로는 not_visual → 원본 레이블의 한계. 정확한 40건으로 visual retrieval 성능을 측정.

### 2.3 40건의 도메인 분포

| 도메인 | 건수 |
|--------|------|
| 사회 | 10 |
| 교육 | 9 |
| 환경 | 6 |
| 외교 | 5 |
| 금융 | 5 |
| 산업 | 4 |
| 기타 | 1 |

## 3. 인터랙티브 HTML 리포트 (E-2)

### 3.1 리포트 구성

각 케이스에 대해:
- **쿼리 텍스트**: 검색 질문
- **GT 페이지 이미지**: 정답 페이지의 전체 이미지 (base64 임베딩)
- **Region crop 이미지**: 해당 페이지의 image/chart crop (base64 임베딩)
- **3방식 Rank 비교**: text / region / caption 각각의 검색 순위

시각적 성능 요약:
- Bar chart: 3방식별 R@K 비교
- Heatmap table: 케이스 × 방식 Hit/Miss 매트릭스
- 7개 카테고리별 그룹핑

### 3.2 기술 구현

- 페이지 이미지와 region crop 이미지를 base64로 인코딩하여 HTML에 직접 삽입 (단일 HTML 파일로 자급)
- JavaScript로 카테고리 필터링, 케이스 탐색
- 파일 크기: 3.2MB (40건의 이미지 포함)

### 3.3 리포트 활용

- D-2의 Region crop 매칭 버그를 이 리포트를 통해 발견 (엉뚱한 이미지가 매칭된 케이스 시각적 확인)
- E-3의 카테고리별 정성 분석이 이 리포트를 기반으로 수행

## 4. 카테고리별 정성 분석 (E-3)

### 4.1 7개 Hit/Miss 패턴

Qwen3-VL-8B R@10 기준 Hit(≤10) / Miss(>10)로 분류:

| 패턴 | 건수 | 비율 | 특징 |
|------|------|------|------|
| **Region만 Hit** | 14 | 35.0% | 구조도/프로세스 다이어그램/조직도. 캡션이 구조적 의미를 전달 못함 |
| **3방식 모두 Hit** | 9 | 22.5% | 표준 차트/표. 정보가 다양한 채널에 중복 존재 |
| **Region + Caption Hit** | 3 | 7.5% | 수치 비교 차트. 캡션이 축/범례/수치를 잘 전달 |
| **Region + Text Hit** | 4 | 10.0% | 텍스트 키워드 충분하나 캡션이 핵심 수치 누락 |
| **Caption만 Hit** | 3 | 7.5% | 지도/전략배치도. 캡션만이 의미적 맥락을 텍스트로 변환 |
| **Text + Caption Hit** | 1 | 2.5% | 키워드 기반 매칭 유효 |
| **모두 Miss** | 6 | 15.0% | 복합 정보 종합 필요. 단일 임베딩으로 커버 불가 |

Region이 관여하는 Hit: **30건 (75.0%)** → 멀티모달 임베딩의 핵심적 역할.

### 4.2 핵심 인사이트

**Region만 Hit (14건)**: 12건이 rg_rank=1. 시각적 구조(화살표, 배치, 관계도)가 담긴 이미지는 cross-modal 멀티모달 임베딩이 압도적. 캡션이 복잡한 다이어그램의 구조적 의미를 텍스트로 전달하는 데 실패하는 전형적 케이스.

**Caption만 Hit (3건)**: 지도/전략도처럼 "보이는 것"보다 "의미하는 것"이 중요한 이미지에서는 캡션이 유일한 검색 경로. 멀티모달 임베딩도 추상적 이미지에서는 의미적 매칭에 실패. 캡션 없으면 완전한 Miss.

**모두 Miss (6건)**: rg_rank가 15~1,053으로 "가까이 갔으나 Hit에 미치지 못한" 케이스 다수. 쿼리가 여러 이미지/페이지에 걸친 정보를 요구.

### 4.3 도메인별 패턴

| 도메인 | 건수 | Region Hit율 | Caption Hit율 | 모두 Miss |
|--------|------|-------------|-------------|----------|
| 교육·산업 | 13 | 89~100% | 33~50% | 0% |
| 외교·환경 | 11 | 40~50% | 50~60% | 20~33% |
| 금융·사회 | 15 | 80% | 20~40% | 20% |

- **교육·산업**: Region 기반 검색이 매우 효과적 (구조도/차트 중심)
- **외교·환경**: Caption 의존도 높음 (지도/전략도/추상적 이미지)

### 4.4 검색 전략 설계 시사점

| 검색 방식 | 강점 이미지 유형 | 약점 이미지 유형 |
|----------|----------------|----------------|
| **Region (멀티모달)** | 구조도, 프로세스 다이어그램, 표준 차트 | 지도, 전략도, 추상적 이미지 |
| **Caption (텍스트 변환)** | 지도, 전략도, 수치 데이터 차트 | 복잡한 다이어그램, 관계도 |
| **Text (OCR)** | 키워드 명시적 포함 문서 | 시각 정보 의존 쿼리 |

Region + Caption 결합이 필수: 단독으로는 각각 75.0%, 40.0%이나 결합 시 80.0%.
